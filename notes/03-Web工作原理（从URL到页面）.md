# 📖 概念笔记：Web工作原理（从URL到页面）

> **一句话**：浏览器输入URL → DNS查IP → TCP三次握手 → HTTP请求 → 服务器处理 → 返回HTML → 浏览器渲染
> **Day 4实操**：打开百度F12，观察Network面板的每一步

---

## 🌐 全链路（7步）

```
你输入 baidu.com 按下回车
        │
        ▼
┌─ 1. DNS解析 ─────────────────────────────────┐
│  浏览器问DNS服务器："baidu.com的IP是多少？"   │
│  DNS回答："110.242.68.66"                    │
│  类比：查电话本，把名字翻译成电话号码         │
└──────────────────────────────────────────────┘
        │
        ▼
┌─ 2. TCP三次握手 ────────────────────────────┐
│  你的电脑："喂，在家吗？"（SYN）             │
│  服务器："在，你确定要聊？"（SYN-ACK）       │
│  你的电脑："确定！"（ACK）                   │
│  类比：打电话，先确认对方接了才开始说正事     │
└──────────────────────────────────────────────┘
        │
        ▼
┌─ 3. HTTP请求 ───────────────────────────────┐
│  你的浏览器发送：                            │
│  GET / HTTP/1.1                             │
│  Host: www.baidu.com                        │
│  User-Agent: Chrome/...                     │
│  类比：你对着电话说"喂，我要看首页"           │
└──────────────────────────────────────────────┘
        │
        ▼
┌─ 4. 服务器处理 ─────────────────────────────┐
│  百度服务器收到请求 → 查数据库 → 拼HTML页面  │
│  类比：接电话的人去翻档案，找出你要的信息     │
└──────────────────────────────────────────────┘
        │
        ▼
┌─ 5. HTTP响应 ───────────────────────────────┐
│  服务器返回：                                │
│  HTTP/1.1 200 OK                            │
│  Content-Type: text/html                     │
│  <html>...百度首页的HTML代码...</html>       │
│  类比：对方说"好的！这是你要的东西"           │
└──────────────────────────────────────────────┘
        │
        ▼
┌─ 6. 浏览器渲染 ─────────────────────────────┐
│  浏览器解析HTML → 构建DOM树 → 计算CSS样式    │
│  → 布局 → 绘制 → 你看到百度首页              │
│  类比：你收到一封信，拆开、读、理解内容       │
└──────────────────────────────────────────────┘
```

## 📊 F12 Network面板速查

### 关键字段

| 字段 | 含义 | 测试怎么用 |
|:---|:---|:---|
| **Request URL** | 请求发给了谁 | 确认API地址是否正确 |
| **Request Method** | GET/POST/PUT/DELETE | 确认请求类型对不对 |
| **Status Code** | 服务器返回的状态 | 200=成功, 404=找不到, 500=服务器炸了 |
| **Response Body** | 服务器返回的内容 | 看返回的数据对不对 |
| **Request Headers** | 请求带了什么附加信息 | 看Authorization/Content-Type等 |

### 常见状态码速记

| 状态码 | 含义 | 记忆方法 |
|:---|:---|:---|
| 200 | OK，成功 | 😊 一切正常 |
| 301/302 | 重定向 | 🔀 搬新家了，自动跳转 |
| 400 | Bad Request | 🙅 你说的我听不懂 |
| 401 | Unauthorized | 🔑 你没带钥匙（没登录/Key错了） |
| 403 | Forbidden | 🚫 你有钥匙但没权限进这个房间 |
| 404 | Not Found | 👻 你要找的东西不存在 |
| 500 | Internal Server Error| 💥 服务器自己出bug了 |
| 502 | Bad Gateway | 🌉 网关挂了 |
| 503 | Service Unavailable | 🛑 服务暂时不可用 |

## 🧪 测试角度：Agent的Web工具失败时

当Agent告诉你"网页打不开"时，你需要三层排查：

```
第1层：请求发出去了吗？
  → F12 → Network → 看有没有请求记录
  → 没有 = 请求根本没发出去（前端/Agent问题）

第2层：服务器回了吗？回的是什么？
  → 有请求记录 → 看Status Code
  → 4xx = 你的问题（权限、参数、地址错）
  → 5xx = 服务器的问题

第3层：返回的内容对吗？
  → 200 OK → 看Response Body
  → 内容对 = 没问题
  → 内容不对 = 业务逻辑问题
```
